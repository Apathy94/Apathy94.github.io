前言

一直想搭建自己的博客，在博客上记录一些东西，一方面记录自己的学习痕迹，另一方面督促自己输出总结。开始了解的博客都是第三方平台，例如WordPress，简书，还有自己买服务器，自己写前端等等方式，不过这个学习成本太高，还是等我慢慢学习之后，再去尝试吧。

一次偶然的机会，我看到了用Github Page + Hexo搭建个人博客的方式，正好最近把论文投出了，难得有空余时间，那就来实践一下～

准备环境

搭建Hexo的过程中需要NodeJS和git，首先需要在电脑上准备安装好环境。我使用的平台是Mac，我使用了homebrew进行安装。

git，一个分布式版本控制系统，用于项目的版本控制管理，作者是 Linux 之父。如果 Git 还不熟悉可以参考廖雪峰大神的 Git 教程。

安装完毕，可以在命令行工具中输入相应命令进行验证，若出现版本号，则证明安装成功

注册Github

若你还没有Github，可以先去自行注册一个。
现在你已经又一个Github账号，那么点击 Start project 或者下面的 new repository 创建一个新的仓库。

[注意]Github 仅能使用一个同名仓库的代码托管一个静态站点。例如我的Github账户名是Apathy94，那么我的仓库名必须是Apathy94.github.io。


因为我已经创建过了，所以这边会有红色感叹号。

配置SSH Key

要使用 git 工具首先要配置一下SSH key，为部署本地博客到 Github 做准备。

打开命令行输入cd ~/.ssh如果没报错或者提示什么的说明就是以前生成过的，直接使用 cat ~/.ssh/id_rsa.pub命令就是可以查看本机上的 SSH key 了。
但是，我这里认为还是觉得需要重新配置一下比较好

生成密钥 SSH key

按照提示完成三次回车，即可生成 ssh key。通过查看 ~/.ssh/id_rsa.pub 文件内容，获取到你的 SSH key

输入eval "$(ssh-agent -s)"，添加密钥到ssh-agent。

再输入ssh-add ~/.ssh/id_rsa，添加生成的SSH key到ssh-agent。

打开github，在头像下面点击settings，再点击SSH and GPG keys，新建一个SSH，名字随便。

输入ssh -T git@github.com，如果如下图所示，出现你的用户名，那就成功了。

部署前最好能先执行一下 hexo clean 命令，清除缓存文件 (db.json) 和已生成的静态文件 (public)。在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

另外，如果你的文章暂时不发布可以先保存在草稿里面。生成草稿的方法和文章差不多 hexo new draft "文章标题"，生成后会在 /source/_drafts 里看到你的草稿文章。当你想发布时只要执行 publish 命令即可发布到博客。

总结

到这里，Hexo+Github的个人博客就搭建好了，现在你就可以在这里记录你自己的学习生活了。当然后面我还会继续对其进行优化，然后加一些其他功能，等我折腾好了，我在记录一下
